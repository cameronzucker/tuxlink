## Deliverable
The running Tauri app exposes a UDS MCP endpoint with ONE inert tool (`server_info`), and the
`tuxlink-mcp` stdio shim pipes a client's stdio to it. End-to-end `claude mcp` works against the
standalone testserver (tier-2) and the real app (tier-3). Proves the spine reaches live
`EgressGuard` state.

## Research-resolved facts (verified vs origin/main + docs.rs, 2026-06-26 — bd notes on cvx84.1)
- **rmcp UDS transport = feature `transport-async-rw`** (NOT `transport-io`, which is stdio-only).
  Blanket `impl IntoTransport<RoleServer,_,_> for (R, W)`; call
  `router.serve((read_half, write_half)).await?; running.waiting().await?` where
  `(read_half, write_half) = UnixStream::into_split()` (`OwnedReadHalf`/`OwnedWriteHalf`, both
  `'static`). App rmcp features: `["server", "macros", "transport-async-rw"]`.
- **The shim needs NO rmcp** — it is a dumb `tokio::io::copy` bidirectional byte-pump (~40 lines).
- **rmcp latest 0.8.x = 0.8.5.** MSRV risk vs project 1.75 is **CI-verified** (push draft, let the
  cold compile answer); never a Pi cold build. If rmcp forces an MSRV bump → STOP, escalate (it is
  an operator decision, not a silent edit).
- `#[tool]`/`#[tool_router]` need `serde` + `serde_json` as DIRECT deps (already present).
- **EgressGuard is on origin/main and `.manage()`-wired** as `Arc<EgressGuard>` (lib.rs:655);
  `is_tainted()` + `armed_remaining()` are the `server_info` reads. No bd dep on bd-7dwqa.
- AppState is ~9 separate `.manage()` handles; the MCP router bundle clones the handles it needs.
- Adapter pattern to mirror: core fn + factory seam + thin `#[tauri::command]` + inline
  `#[cfg(test)]` (e.g. `modem_ardop_connect_gated_with_factory`, modem_commands.rs:169).

### Task 3 — UDS transport (`transport_uds.rs`): single-caller accept loop, 0600, unlink-on-exit
- File: new `src-tauri/src/mcp/transport_uds.rs`.
- `UnixListener::bind(sock_path)`; `chmod 0600` the socket; refuse to bind if the parent runtime dir
  is group/world-writable (return an error, log it). Single-caller: accept ONE live connection at a
  time (design Q6 — makes caller identity trivial for audit); while one connection is served, do not
  accept a second (or accept-and-immediately-close the second with a clear error). On accept:
  `let (rh, wh) = stream.into_split(); router.clone().serve((rh, wh)).await?; running.waiting().await`.
  Unlink the socket path on shutdown (Drop guard or explicit cleanup).
- Sock path: from arg/env `TUXLINK_MCP_SOCK`, else a per-user runtime dir
  (`$XDG_RUNTIME_DIR/tuxlink/mcp.sock` or platform equivalent).
- TDD: a tokio test that binds the listener on a tempdir path, connects a client `UnixStream`,
  and asserts the 0600 perms + that a second concurrent connect is rejected. (No rmcp round-trip in
  the unit test — that is tier-2.)
- Pitfall watch: stale socket file from a prior crash — bind should remove a stale socket it owns
  before binding (but NOT a socket another live instance owns; check liveness or use the runtime
  dir + single-instance assumption). Document the choice.

### Task 5 — `tuxlink-mcp` stdio shim (dumb byte-pump, NO rmcp)
- Files: `src-tauri/tuxlink-mcp/Cargo.toml` (deps: `tokio` with `["net","io-util","macros","rt-multi-thread"]`

- Read sock path from argv[1] or env `TUXLINK_MCP_SOCK`. Connect `UnixStream`. Split both the
  socket and stdio; `tokio::io::copy` stdin->socket and socket->stdout concurrently

- TDD: a test that pipes bytes through a loopback `UnixListener` echo and asserts round-trip. The
  shim crate is small and Pi-buildable, so this test runs locally.
- DoD: `claude mcp add tuxlink -- <path>/tuxlink-mcp` (with TUXLINK_MCP_SOCK set) connects to a
  listening socket.
