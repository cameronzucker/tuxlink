# SBC Codec (UV-Pro SSTV) Implementation Plan — tuxlink-vgvn

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development or executing-plans. TDD against the golden vectors below.

**Goal:** A pure-Rust SBC codec implementing the transport's `SbcCodec` trait: decode (radio → PCM) and encode (PCM → radio). The wire payload of `AudioData` frames is SBC; this is the layer between SSTV-PCM and the transport.

**Architecture:** **Decode = `mini_sbc`** (pure-Rust crate, MIT/Apache, GPL-3 compatible) — VERIFIED this session (see below). **Encode = a from-scratch pure-Rust port** (no pure-Rust SBC encoder exists on crates.io), validated **off-air** via the operator's loopback insight: encode → decode (mini_sbc / ffmpeg) → compare PCM. Develop the encoder in the **standalone fast-iteration crate** (`dev/scratch/sbc-proto/`, builds in ~3s — the tauri cold-build precludes iterating in-tree) against the golden vectors, then port the verified module into `src-tauri`.

**Params (pinned, = benlink + the UV-Pro):** 32 kHz, **mono**, **8 subbands**, **16 blocks**, **bitpool 16**, **Loudness** allocation, `msbc=false`. Confirmed by the ffmpeg golden vector's frame header `9c 71 10` (sync / 32k+16blk+mono+Loudness+8sb / bitpool16). Frame = 40 bytes, 128 PCM samples/frame.

## VERIFIED THIS SESSION (opossum-yew-juniper, 2026-06-13)

- **Golden vectors generated** via ffmpeg sbc encoder (`dev/tools/gen-sbc-golden-vectors.sh`): a deterministic 1 kHz sine, 4096 samples → `sine1k_32k_mono.{pcm,sbc}` (committed under `src-tauri/.../audio/testdata/`). 32 frames × 40 bytes = 1280 SBC bytes.
- **`mini_sbc` decode VERIFIED** in the standalone crate: decode the golden SBC → PCM, MAE **6.7** (0.04% of peak 16383) after accounting for the **137-sample SBC synthesis-filterbank startup delay**. So `mini_sbc` is a faithful decode oracle and the decode path is proven. API: `SBCHeader::decode(&mut &[u8])` → `FrameDecoder::new(&h, &mut FilterState::<1,8>::new(), &mut data)`, iterate → `[[i16; 8]; 1]` PCM per block.
- **The 137-sample delay matters for tests:** compare ref[i] vs out[i+137] (or trim), not ref[i] vs out[i]. A naive aligned compare shows ~98% MAE (garbage) and is a false negative.

## Task 2: Encode port (the hard half — bit-exact-ish, validated by round-trip)

**Reference for coefficients:** `mini_sbc`'s `table.rs` (the synthesis prototype window; the analysis window is the same prototype, applied in the analysis direction) — extract the 8-subband proto coefficients from there or the A2DP spec. Develop in `dev/scratch/sbc-proto/` first.

The SBC encoder steps (per the A2DP spec / FFmpeg `sbcenc.c`):
- [ ] **Analysis filterbank** (8-subband polyphase): per block, push 8 new samples into a 80-tap window buffer, apply the proto window + the 8×16 cosine-modulation matrix → 8 subband samples.
- [ ] **Scale factors:** per subband, `scale_factor = bits needed to represent max|sample|`.
- [ ] **Bit allocation (Loudness mode):** the SBC Loudness allocation algorithm using the per-subband offset table for 8 subbands @ 32 kHz, distributing `bitpool=16` bits. MUST match what the decoder expects (the decoder derives the same allocation from the scale factors + bitpool — so encode + decode must agree). Cross-check: our encoded frame's implied allocation must let `mini_sbc` decode it.
- [ ] **Quantize** each subband sample to its allocated bits.
- [ ] **Bitstream pack:** sync `0x9C`, header byte (`0x71` for our params), bitpool byte (`0x10`), **CRC-8** over the header+scale-factor region (poly per spec — `mini_sbc/src/crc.rs` is the reference), then scale factors (4 bits each) + quantized samples.
- [ ] **Round-trip test (the off-air oracle):** `encode(pcm)` → `mini_sbc decode` → compare to `pcm` (137-sample delay), assert MAE below a fidelity threshold. ALSO cross-decode via ffmpeg (`ffmpeg -f sbc -i ours.sbc -f s16le -`) to confirm a second independent decoder accepts our stream. Structural check: our frame header == `9c 71 10`.

## Task 3: Assemble the `SbcCodec` trait impl + wire into the transport

- [ ] `struct UvproSbcCodec` implementing `SbcCodec` (`encode`/`decode`) from the transport's `audio/codec.rs` seam. `encode(pcm_bytes)` → frame the PCM into 128-sample blocks → SBC bytes; `decode(sbc)` → PCM bytes (s16le). Handle partial blocks (pad or buffer).
- [ ] The transport's `UvproSession::open_audio(codec)` already takes `Arc<dyn SbcCodec>` (wired in PR #671) — inject `UvproSbcCodec` from the eventual Tauri command / UI (tuxlink-yfyn).

## Notes / pitfalls
- mono/8-subband/bitpool-16 is the radio's fixed format; don't over-generalize the codec to all SBC modes (YAGNI) — but DO assert the decoded header matches, and reject mismatches.
- The synthesis/analysis delay (137 samples) is inherent; the SSTV decoder (tuxlink-st5n) must tolerate the codec's group delay (it's an STFT decode, robust to a fixed offset).
- Codex adversarial review of the finished encoder is warranted (it produces transmitted audio) — run it once Codex quota resets.
