Modem console placements — compact-in-dock vs full-overlay

v0.5+ surface. Showing both placements satisfies the two-mode operator workflow you identified.

Operator-mode design call captured

  • Mode 1 (high-volume / good conditions): reduced visual clutter, more space for composing/tracking. Compact dock with glanceable modem health.
  • Mode 2 (marginal / QRP / troubleshooting): full diagnostic visibility while actively adjusting band / power / target / antenna. Full-overlay pane with constellation, throughput chart, all gauges.
  • Same View → Show Modem Console toggle, three-state: Off · Compact (in dock) · Full (overlay pane). Operator switches mode based on conditions; tool serves both.
  • Aesthetic tension noted: instrument-panel green-on-dark for the modem-specific elements; Mail.app polish for everything else. The transition zone is the dock/pane border — clean.

Full pane contents: throughput chart (60s history, color-shifts when SNR drops), constellation diagram (QPSK scatter, warning dots for outliers), four big gauges (S/N, VU, CPU, TX delay), full protocol state lights (9-wide), comprehensive status strip (peer / freq / power / bandwidth / RX-TX bytes), troubleshooting action buttons (Change band / Increase power / Try different gateway / Modem settings… / Disconnect). The reading pane is hidden; rows pane stays so user can still navigate messages.
Mode-2 operator scenario: Cameron is on 10W QRP in a deployment, S/N was solid at +18 dB but the link is degrading — now +6 dB and bandwidth has auto-renegotiated to 500 Hz NARROW. Throughput dropped from 14k to 680 bps. The constellation has spread out (visible scatter, some yellow outliers). He needs to decide: bump power, change band, try a different gateway, or accept the slow link. The full-overlay pane gives him every signal he needs without context-switching to a separate VARA window.

v0.5+ modem-console placement supports both operator modes. Same toggle (View → Show Modem Console), three states (Off · Compact · Full). Compact lives in the right dock; Full replaces the reading pane.

This is a v0.5+ surface — not in v0.0.1 scope. v0.0.1 dock contents stay light (session timer, outbox queue, last sessions) with the modem section labeled "v0.5+" in mocks-v2's stub area.
