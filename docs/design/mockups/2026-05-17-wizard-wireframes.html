Same dark-mode polish as the locked v0.0.1 direction. Branching wizard tree: CMS-optional, offline-friendly, tactical-callsign-friendly, test-send-never-blocking.

Branching wizard tree — design call captured from this brainstorm

  • Step 1 asks intent, not "do you have an account?" — "Will this installation connect to the Winlink CMS?" with two paths from the same first screen.
  • CMS-yes path: Step 1 → Step 2 (credentials, with register link inline) → Step 3 (test-send, never blocking) → inbox.
  • CMS-no path: Step 1 → Step 2-offline (minimal identity, callsign optional) → inbox. Two-screen wizard for the lab / training / offline-deployment operator.
  • Test-send is informational, not gating — every substate (idle / sending / success / failed) has a path to the inbox. Plan Task 11 amendment required.
  • Validator loosened to permissive — non-empty + no whitespace + ≤32 chars. Accepts W4PHS, EOC-1, BAOFENG-FM-01, EMCOMM-NET, K7XYZ/P, etc. CMS authoritates server-side.
  • Aux/tactical callsigns: wizard takes ONE primary callsign; aux callsigns managed later via Tools → Settings → Callsigns (deferred from v0.0.1).
  • Grid square: manual entry in v0.0.1; Auto-detect from GPS affordance disabled with "v0.1" badge (matches Express's Update grid square from GPS=True pattern).

Verify your setup

Sends a short test message to SERVICE@winlink.org (Winlink's autoresponder) and waits for the reply. Confirms your credentials, your network, and the path to CMS. Skipping is fine — you can run this anytime from Session → Test send.

Transport: prefers CMS-SSL (port 8773, TLS) when reachable; falls back to Telnet (port 8772) otherwise. Change in Settings → Connection → Transport. (Principle: never auto-select a security-relevant transport silently.)

Substate 1 · Idle (initial)
▸

Ready to verify

One outbound + one inbound message to SERVICE@winlink.org. Takes 10-30 seconds depending on your network.

Substate 2 · Sending

Connecting to cms.winlink.org…

Posted message to outbox · Triggered telnet connection · Waiting for SERVICE reply (timeout 30s).

Substate 3 · Success
✓

Verified — reply received in 8s

Connected via CMS-SSL gateway 1235-2.cms.winlink.org. Your test message and the autoresponder's reply are in your inbox.

Substate 4 · Failed
!

Couldn't reach CMS within 30s

Your credentials are saved. Most likely: no internet, firewall blocking :8773, or CMS busy. Not a setup error — you can still use Tuxlink.

All four substates lead to the inbox — verification is informational, never blocking
Design notes: All 4 substates shown in one mock so the design intent — "never blocking" — is visible at a glance Failed state uses warning yellow not error red — it's not catastrophic, just informational Failed state names the most likely causes inline so the operator can act without a help doc Step note "Optional — you can skip and open the inbox now" reinforces the rule at every visit Real implementation shows ONE substate at a time; this mock is for design-review only In implementation: idle → sending happens on button click; sending → success/failed happens on server response or 30s timeout

Plan amendments queued for the design doc (from these wireframes)

  • Task 9 (Step 1) — change "Do you have a Winlink account?" question to "Will this installation connect to the Winlink CMS?" with branching to credentials-or-offline. Register-account link moves to Step 2 as inline reference, not a sub-step.
  • Task 10 (Step 2 credentials) — loosen callsign validator from ^[A-Z0-9]{3,6}$ to permissive (non-empty + no whitespace + ≤32 chars). Add "Save credentials and skip verification" button. Update field labels to match wireframes.
  • Task 11 (Step 3 test-send) — make test-send informational not blocking. Add 4-substate UI (idle / sending / success / failed) with paths to inbox from every state. Add Session → Test send menu item so test can be re-run later.
  • NEW Task 11.5 — offline path — Step 2-offline component, minimal identity form, no test-send, advances directly to inbox.
  • Schema change — Config needs connect_to_cms: bool field plus optional identifier: String (for offline path). Callsign field becomes optional when connect_to_cms: false.
  • NEW menu item — Session → Test send (post-wizard re-run). Tools → Settings → Connection (toggle CMS-on/off post-wizard).
  • NEW v0.1+ — MPS (Message Pickup Station) registration UI — when v0.1 brings radio-only / Hybrid Network mode, operators need to register 1-3 MPSs (per Radio_Only_Winlink.pdf, ARSFI). New menu: Tools → Connection → Message Pickup Stations. Not in v0.0.1 scope (telnet-only doesn't use MPSs); flagged now so the menu structure has the slot.
